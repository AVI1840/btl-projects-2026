# Project Structure

```
├── index.html                  # Main portfolio page (source of truth)
├── build-internal.js           # Generates html/index.html for intranet
├── server.js                   # Local dev server (port 3456)
├── html/
│   ├── index.html              # AUTO-GENERATED — do not edit
│   ├── btl-logo.png            # Logo assets
│   ├── btl-emblem.png
│   ├── btl-logo.svg
│   ├── btl_page.html           # BTL official page template (gitignored)
│   ├── feedback-dashboard.html # Feedback analytics dashboard
│   ├── <tool-name>.html        # One-pager per tool (e.g. arnak-zchuyot.html)
│   └── redirects/
│       └── <tool-name>.html    # Intranet redirect stubs (→ snifim.blroot)
├── word/
│   ├── <NN>-<tool-name>.docx   # Word documents per tool (numbered)
│   ├── <NN>-<tool-name>.md     # Markdown source for some docs
│   ├── build-docx.js           # DOCX generation script
│   └── build-docx-v2.js        # Updated DOCX generation
├── scripts/
│   ├── build-internal.js       # (duplicate/alternate of root build-internal.js)
│   └── update-feedback.js      # Batch feedback system updater
└── .kiro/
    └── steering/               # AI assistant steering rules
```

## Naming Conventions
- Tool HTML pages: kebab-case matching the tool name (e.g. `dashboard-gamlaut.html`)
- Word docs: numbered prefix `NN-tool-name.docx` (e.g. `03-dashboard-gamlaut.docx`)
- Redirect files mirror the tool HTML filename in `html/redirects/`

## Important Rules
- `html/index.html` is auto-generated by `build-internal.js` — always edit root `index.html` instead
- One-pager HTML files are self-contained (no external CSS/JS dependencies beyond Google Fonts)
- Each one-pager includes an embedded feedback system with localStorage + Google Sheets integration
